# Rock, Paper, Scissors - Logic Case Study

This repository contains my submission for a **programming logic class project** where the goal was to design the logic for a Rock, Paper, Scissors game.

The primary focus of the assignment was algorithm design using pseudocode and flowcharts. I also implemented the program in Python to demonstrate how the logic translates into working code.

## Assignment Brief

Design a program that allows a user to play **Rock, Paper, Scissors** against the computer.

The program must follow this logic:

1. When the program begins, the computer generates a random number between 1 and 3:
   - `1` → Rock  
   - `2` → Paper  
   - `3` → Scissors  

2. The user enters their choice:
   - `rock`
   - `paper`
   - `scissors`

3. The computer’s choice is revealed.

4. The program determines the winner according to the following rules:

- Rock beats scissors (rock smashes scissors)
- Scissors beat paper (scissors cut paper)
- Paper beats rock (paper wraps rock)

5. If both players choose the same option, the game must repeat until a winner is determined.
